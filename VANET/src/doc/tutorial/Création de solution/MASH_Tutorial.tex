\section{Cas d'étude : les agents horloges}

Le cas d'étude du tutoriel sera une application mêlant deux types d'agents. Des agents horloges (\texttt{Horloge}) et des agents utilisateurs (\texttt{Utilisateur}).

Les agents \texttt{Horloge} diffusent toutes les 10 secondes l'heure à tous les agents qui sont dans leur portée d'émission. Ces agents peuvent tout de même répondre à des requêtes venant des utilisateurs. Elles peuvent décliner leur identité et l'heure (ces deux messages ne sont pas destinés à tous les agents voisins mais uniquement à destination de l'initiateur de la requête).

Les agents \texttt{Utilisateur} remercient les horloges lorsqu'elles indiquent l'heure en déclinant leur identité. Il émettent une requête toutes les 25secondes  soit pour demander l'heure aux horloges voisines (probabilité de $\frac{4}{5}$) soit pour connaître l'identité des horloges voisines et ne demander l'heure qu'à l'une d'entre elles (probabilité de $\frac{1}{5}$). 

\section{Création d'une solution}

Pour créer une solution il est nécessaire de créer dans le package \texttt{Simulation.Solution.Custom} une classe héritant de la classe \texttt{Simulation.Solution.SolutionItem}.

Nous allons créer la classe \texttt{HorlogeSolution}. Il est a remarquer que :
\begin{itemize}
\item seul l'instance de la classe \texttt{Simulation.Solutions.Solution} passé en paramètre de la méthode \texttt{setSolution} change d'une solution à une autre.
\item il est nécessaire de donner une référence vers les classes des objets et agents que peut contenir le système multi-agents. 
\end{itemize}

Dans la solution proposée, nous supposerons que les comportements des agents (\texttt{Horloge}) et des agents (\texttt{Utilisateur}) seront codés dans une même classe (la classe \texttt{Simulation.Solutions.Custom.AgentHorloge.AgentHorlogeUtilisateur}).

\begin{verbatim}
package Simulation.Solutions.Custom;

import Simulation.Solutions.Solution;
import Simulation.Solutions.SolutionDescriber;
import Simulation.Solutions.SolutionDescriberException;
import Simulation.Solutions.SolutionException;
import Simulation.Solutions.Custom.AgentHorloge.AgentHorlogeUtilisateur;

/**
 * Horloge solution
 * @author Jean-Paul Jamont
 */
public class HorlogeSolution extends SolutionItem
{
	public HorlogeSolution()
	{
		super();

		try
		{
			super.setSolution(
					new Solution(
							new SolutionDescriber(
									"Jean-Paul Jamont" 					/* nom de l'auteurs de la solution */,
									"jean-paul.jamont@iut-valence.fr" 	/* adresse mail du correspondant*/,
									"Universite Pierre Mendes France" 	/* établissement de rattachement*/,
									"51 rue barthelemy de Laffemas, 26000 Valence" /* adresse */,
									"France" 							/* pays */,
									"HORLOGE - Agent Horloge" 			/* Dénomination */,
									"1a" 								/* version */,
									"Implementation of a user/clock toy problem" /*Description */
							), 
							new AgentHorlogeUtilisateur(null,0,1f,0).getClass() /* référence vers l'agent */
					)
			);
		} 
		catch(SolutionException e)
		{
			e.printStackTrace();
		}
		catch(SolutionDescriberException e)
		{
			e.printStackTrace();
		}
	}
}

\end{verbatim}
